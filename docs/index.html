Deterministic race-strategy engine

Does the undercut work?

A physics-light F1 simulator that turns one strategic question into a number. It models lap time from base pace + fuel burn + tyre wear, then advances the field forward — pit-now versus stay-out — and returns a verdict.

02 / PACE MODEL

predicted_lap_time

Every lap time the engine produces is one clean sum of decoupled effects — no double-counting, each term fit from real lap data.

predicted_lap_time = base_pace[driver] + fuel_effect(lap) + deg_effect(tyre_age, compound)
base_pace[driver]

Each driver's reference lap — the median of their clean green-flag laps. The anchor every correction is measured against.

median clean lap
fuel_effect(lap)

Cars get lighter as fuel burns, so laps get faster. Expressed relative to mid-race fuel so it never double-counts the base.

~0.035 s/kg burned
deg_effect(age)

Tyre wear, fuel removed: a per-compound linear rate applied against a reference age. Soft falls off fast, hard holds on.

fuel-decoupled slope
03 / SIGNAL

Tyre degradation, fuel removed

Fuel-correct every lap, fit a linear slope per stint, average across the field. What's left is pure wear — seconds lost per lap of tyre age, per compound.

04 / ARCHITECTURE

One engine, everything composes

A clean data layer feeds three decoupled models, which feed one pure forward-simulation engine. Every strategic question is just that engine, called differently.

DATA LAYER
clean · loader
Build one canonical schema: flag out-laps, in-laps, SC/VSC and inaccurate laps so only honest green-flag pace survives.
MODELS
pace · fuel · tyre_deg
Three modules, each owning one effect, injected — never importing each other. Decoupled by construction.
STATE
CarState
Per-car dynamic state: tyre age, compound, cumulative time, pit lap. The quantity that evolves lap to lap.
↓
THE CORE · PURE FUNCTION
forward_sim()
Advance the whole field N laps. Position falls out of cumulative time — sort-order, nothing more. Two cars or twenty: same loop.
↓
undercut_verdict()
Run the engine twice — pit now vs stay out — diff the gap to the rival. Does it work, and by how much?
backtest
Run it with the real pit lap and compare against what actually happened on track. Does the model hold up?
monte carlo
Run it many times with noise on the inputs. Turn a single verdict into a probability the undercut lands.
